Stage 0: Getting Started with HTML

The Basics of the Web and HTML

Introduction

World Wide Web also known as WWW, an advanced information retrieval system that supports many kinds of information such as pictures, videos, graphs, fonts and many more. WWW is based on HyperText Markup Language, commonly referred as HTML, the standard markup language used to create web pages. It is written in the form of HTML elements consisting of tags enclosed in angle brackets, e.g. <tagname>.

The internet uses HTTP to communicate with you, your browser and servers.
For example:
A person logs onto their computer and runs a type of program called a browser. The browser makes requests of servers through the internet using a protocol called HTTP and the server will respond with information that your browser interprets and displays.

Browser - A program (Chrome, FireFox, Safari) that runs on your computer to display files found on the web.
Internet - The world's most largest computer network.
Servers - Computers that host the files that make up the web.
HTTP - The main protocol of the web.

HTTP, shortform for HyperText Transfer Protocol, the set of rules for transferring files (text, graphic images, sound, video, and other multimedia files) on the World Wide Web. As soon as a Web user opens their Web browser, the user is indirectly making use of HTTP.

In computing, a hyperlink is a reference to data that the reader can directly follow either by clicking or by hovering. A hyperlink points to a whole document or to a specific element within a document. Hypertext is text with hyperlinks.

HTML is made up of:

  • Text content - What you see
  • Markup - What it looks like
  • References to other documents - Such as images, videos
  • Link - Shortcut to other pages

Intro to HTML Tags

HTML Elements


Example:
    "<tagname>Contents</tagname>"

<tagname> - Openning tag
Contents - Text to be displayed/effected by the tag.
</tagname> - Closing tag

Reminder: If you do not input the closing tag, the following contents will be formatted as the openning tag.

List of Basics Tags

  • <b> tag specifies bold text
  • <em> tag renders as emphasized text, it specifies italic

HTML Attributes

  • HTML elements can have attributes.
  • Attributes provide additional information about an element
  • Attributes are always specified in the openning tag
  • Attributes come in name/value pairs like: name="value"

1. Making Links

HTML links are defined with the <a> tag. The link address is specified in the href attribute:
    <a href="value">text</a>

"value" refers to the link that you wish to direct to

2. Adding Images

HTML images are defined with the <img> tag.
The alt attribute specifies an alternative text to be used, when an HTML element cannot be displayed.The value of the attribute can be read by "screen readers". This way, someone "listening" to the webpage, i.e. a blind person, can "hear" the element.
The filename of the source (src), and the size of the image (width and height) are all provided as attributes:
    <img src="url" alt="text" width="size" height="size">

Reminder: "<img>" is a void tag, other void tags include "<br>" etc. Void element does not required any closing tag.

HTML Block Elements and Inline Elements

Most HTML elements are defined as block level elements or inline elements:
Block level elements normally start (and end) with a new line, when displayed in a browser.
Examples: <p>, <h1>, <ul>, <table>
Inline elements are normally displayed without line breaks.
Examples: <br>, <b>, <img>, <a>

The HTML <div> element is a block level element that can be used as a container for other HTML elements.
The <div> element has no special meaning. It has no required attributes, but style and class are common.
Because it is a block level element, the browser will display line breaks before and after it.
When used together with CSS, the <div> element can be used to style blocks of content.

The HTML<span> element is an inline element that can be used as a container for text.
The <span> element has no special meaning. It has no required attributes, but style and class are common.
Unlike <div>, which is formatted with line breaks, the <span> element does not have any automatic formatting.
When used together with CSS, the <span> element can be used to style parts of the text.
